이 사이트는 SQL Injection의 원리를 학습하기 위한 교육용 실습 환경입니다. 모든 SQL 처리는 브라우저 내에서만 실행되며 외부 서버에 영향을 주지 않습니다. 허가 없이 실제 시스템에 SQL Injection을 시도하는 것은 불법입니다.

SQL Injection이란?

SQL (Structured Query Language)

데이터베이스에서 데이터를 조회, 삽입, 수정, 삭제하기 위한 언어입니다.

SELECT * FROM users WHERE username = 'admin' AND password = '1234'

SQL Injection의 원리

사용자 입력값이 SQL 쿼리에 직접 삽입될 때, 공격자가 SQL 구문을 조작하여 의도치 않은 쿼리를 실행시키는 공격 기법입니다.

정상적인 입력

username: admin

password: 1234

SELECT * FROM users WHERE username = 'admin' AND password = '1234'

악의적인 입력

username: admin'--

password: (아무거나)

SELECT * FROM users WHERE username = 'admin'--' AND password = '아무거나'

-- 뒤의 모든 내용이 주석 처리되어 비밀번호 검증이 무시됩니다!

실습 1: 로그인 우회 (Authentication Bypass)

시나리오

아래는 웹 사이트의 로그인 페이지입니다. 서버 내부에서는 다음과 같은 SQL 쿼리가 실행됩니다:

SELECT * FROM users WHERE username = '{username}' AND password = '{password}'

목표: 비밀번호를 모르는 상태에서 admin 계정으로 로그인하세요.

힌트

힌트 1: SQL에서 -- 는 한 줄 주석을 의미합니다. 주석 뒤의 모든 내용은 무시됩니다.
힌트 2: username 필드에 작은따옴표(')를 포함시키면 SQL 문자열을 "탈출"할 수 있습니다. admin'--를 시도해보세요.
정답: username에 admin'-- 입력, password는 아무 값이나 입력
완성되는 SQL: SELECT * FROM users WHERE username = 'admin'--' AND password = 'x'
-- 이후는 주석이 되어 비밀번호 검증이 완전히 무시됩니다.

다른 방법: ' OR 1=1-- 를 username에 입력하면 모든 사용자로 로그인됩니다.

힌트 1: UNION SELECT를 사용하면 다른 쿼리의 결과를 합칠 수 있습니다. 단, 원래 쿼리와 컬럼 수가 동일해야 합니다.
힌트 2: 원래 쿼리는 2개 컬럼(name, price)을 반환합니다. 먼저 ' UNION SELECT 'test', 'test'-- 로 UNION이 동작하는지 확인해보세요.
정답: 검색창에 다음을 입력:
' UNION SELECT username, password FROM users--

완성되는 SQL:
SELECT name, price FROM products WHERE name LIKE '%' UNION SELECT username, password FROM users--%'
users 테이블의 데이터가 상품 목록에 함께 표시됩니다.

방어 실습: SQL Injection 막기

어떻게 방어할까?

SQL Injection을 방어하는 가장 효과적인 방법은 Parameterized Query (매개변수화된 쿼리)를 사용하는 것입니다.

취약한 코드

// JavaScript (Node.js) - 위험!
const query = `SELECT * FROM users
  WHERE username = '${username}'
  AND password = '${password}'`;
db.execute(query);

사용자 입력이 SQL에 직접 삽입되어 조작 가능

안전한 코드

// JavaScript (Node.js) - 안전!
const query = `SELECT * FROM users
  WHERE username = ?
  AND password = ?`;
db.execute(query, [username, password]);

? 자리에 값이 안전하게 바인딩되어 SQL 조작 불가

SQL Injection 방어 체크리스트

언어별 안전한 코드 예시

Python (SQLite)

cursor.execute(
    "SELECT * FROM users WHERE username = ? AND password = ?",
    (username, password)
)

Java (JDBC)

PreparedStatement ps = conn.prepareStatement(
    "SELECT * FROM users WHERE username = ? AND password = ?"
);
ps.setString(1, username);
ps.setString(2, password);

PHP (PDO)

$stmt = $pdo->prepare(
    "SELECT * FROM users WHERE username = :user AND password = :pass"
);
$stmt->execute(['user' => $username, 'pass' => $password]);
